# 工作流映射

## 采集工作流

TrendForge 的采集层分三档：

- `rsshub-source`：常规订阅和稳定源，默认优先。
- `browseract-fetcher`：疑难网页和需要浏览器上下文的完整原文补采。
- `mediacrawler-fallback`：中文自媒体平台备用采集器，必须显式启用。

MediaCrawler 不作为默认采集入口；它只在 RSSHub 无结果、BrowserAct 也无法稳定补采，且用户配置允许时触发。

## 公众号工作流

这套工作流适合映射为 `wechat-publisher` 适配器。

核心输入：

- 文章 brief
- Markdown 正文
- 配置文件
- 封面图策略

核心输出：

- 预览 HTML
- 最终 HTML
- 草稿箱结果
- 发布状态记录

它在 TrendForge 里的职责是：

- 接收平台化成稿
- 做格式检查
- 处理封面与正文图片
- 创建公众号草稿

## 小红书工作流

这套工作流适合映射为 `xhs-browser-draft` 适配器。

核心输入：

- 标题
- 正文
- 图片
- 浏览器登录态
- bridge 连接状态

核心输出：

- 发布页填充结果
- 草稿保存结果
- 页面真实状态信号

它在 TrendForge 里的职责是：

- 接收小红书版成稿
- 检查浏览器链路是否可用
- 填充发布页
- 保存到草稿箱

## 对应关系

| TrendForge 能力 | 公众号工作流 | 小红书工作流 |
| --- | --- | --- |
| 内容结构化 | article brief | 标题/正文/图片结构 |
| 平台改写 | Markdown 成稿 | 小红书风格成稿 |
| 图片策略 | 封面 + 正文图 | 图文发布图片 |
| 发布适配器 | API 草稿发布 | 浏览器草稿保存 |
| 成功信号 | 草稿创建成功 | 页面显示草稿已保存 |

## 采集适配器对应关系

| TrendForge 能力 | RSSHub | BrowserAct | MediaCrawler |
| --- | --- | --- | --- |
| 常规订阅 | RSS URL | 不负责 | 不负责 |
| 疑难网页 | 不负责 | 浏览器打开和抽取 | 仅备用 |
| 中文自媒体平台 | 依赖 route | 浏览器补采 | 关键词/详情备用采集 |
| 登录态复用 | 由 RSSHub route 决定 | 复用浏览器上下文 | 复用 CDP/登录态 |
| 默认启用 | 是 | 是 | 否 |
| 触发方式 | 周期订阅 | 验证/全文失败后 | 显式开启后的最后兜底 |
